✅ Security Audit Checklist

Quick reference for code reviews and pre-deployment checks.

🏗️ Architecture

  • ✅ CEI pattern followed in all functions?
  • ✅ Initializers protected against re-init?
  • ✅ Upgrade logic whitelisted/timelocked?
  • ✅ Component storage namespaces unique?

🔢 Arithmetic

  • ✅ No `felt252` used for balance logic?
  • ✅ Integer division precision handled?
  • ✅ Multiplication before division?
  • ✅ Safe integer types (u8-u256) used?

🔑 Access Control

  • ✅ `get_caller_address()` not in constructor?
  • ✅ All admin functions restricted?
  • ✅ L1 handlers validate `from_address`?
  • ✅ Custom AA validates signatures?
